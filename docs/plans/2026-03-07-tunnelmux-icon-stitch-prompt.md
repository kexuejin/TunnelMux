# TunnelMux Icon Stitch Prompt

Use this prompt in Stitch to generate branded icon concepts for TunnelMux.

```text
Create a compact design board for branded app icon exploration for TunnelMux.

Product positioning:
TunnelMux is a developer-first local tunnel control console. It manages local-to-public tunnels, routing, gateway behavior, and service exposure. It should feel reliable, technical, modern, and memorable. It is a desktop developer tool, not a playful app and not a heavy enterprise security brand.

Brand goals:
- high recognition at small sizes
- clear silhouette
- premium desktop-tool feel
- strong brand personality
- suitable for app icon, tray icon, installer icon, and window branding

Visual direction:
- modern, geometric, crisp
- dark-background friendly
- electric blue, cyan, and violet accents
- subtle depth, but not over-rendered
- no mascot, no cartoon, no generic cloud, no generic shield

Concept board layout:
1. Header: “TunnelMux Icon Concepts”
2. Three icon directions labeled A, B, C
3. Each direction shows:
   - large dark-mode icon preview
   - small light-surface preview
   - one-line rationale
4. Footer note: “Designed for app icon, tray icon, installer, and desktop branding”

Concept directions:
- A: tunnel/gateway monogram, strongest recommendation
- B: signal/gateway route mark
- C: portal/mux badge

Semantic cues to explore:
- tunnel / portal
- gateway / ingress
- route merging / multiplexing
- subtle hint of T or M without relying on typography

Output style:
- serious product design review artifact
- not a marketing slide
- balanced spacing, high contrast, clear labels
```
